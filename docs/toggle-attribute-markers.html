CheckBox.js


Toggle Attributes - :markers


This option alternates a destination value between the two checkbox markers. Here, the first marker represents when the checkbox is unchecked while the second marker represents when the checkbox is checked. Whenever the checkbox is unchecked, the first marker's attribute will be assigned the destination value. However, when the checkbox is checked, the value is removed and added to the second marker's attribute. A sample html structure of this is shown below.

HTML JS (GLOBAL) 
<div checkers class="foo" data-switch="bar">
  <div marker style="background-color: red;"></div>
  <div marker style="background-color: green;"></div>
</div>
<input type="checkbox">

<script>
new CheckBox({
  target: '[checkers]',
  assign: {
      attr: ['data-switch', 'class', ':markers']
  }    
});
</script>

From the sample html structure, the "bar" value will be assigned to the class attribute of the first marker only when the checkbox is unchecked and inversely added to the second marker only when the checkbox is checked. Similarly, we can achieve this using the data-assign attribute. An example of this is shown below :

HTML CSS 
<div checkers class="foo" data-switch="bar" data-assign=":markers|data-switch|class">
  <div marker style="background-color: red;"></div>
  <div marker style="background-color: green;"></div>
</div>
<input type="checkbox">

[checkers] {
  width: fit-content;
  border-radius: 100vh;    
  border: dotted 2.5px #514e68; 
  padding: 4px;    
}

[checkers] > [marker] {
  width: 60px;
  height: 2px;    
  background-color: transparent;
  transition: width .5s ease-in-out, 
              background-color .5s ease-in-out;
}

[checkers] > [marker] {
  border-radius: 100vh;
  background-color: yellow;
  margin: 4px;
}

[checkers] > [marker]:first-child {
  width: 30px;
  background-color: red;
}

[checkers] > .bar[marker] {
  width: 120px;                         
  background-color: lime;
}

<script>
new CheckBox({
  target: '[checkers]'
});
</script>
