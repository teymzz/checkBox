CheckBox.js


Advanced Effects


The checkbox plugin supports extended relationships between custom checkboxes. This relationship is one that involves a group of custom checkboxes that are closely or inversely related. Since this relationshiprequires a parent group, the html structure is also slightly different. Here, the parent element is usually defined with an attribute of data-role and the value is usually set as "checkbox-list". This parent element is used to group the custom checkboxes. Once we place the custom checkbox inside this parent element, we can then proceed to define the type of toggle effect we need to achieve. The sample structure for setting up advance effects is shown below:

Relative checkboxes
Copy 
<!--parent checkbox (grand group)-->
<div data-role="checkbox-list">

  <!--first checkbox section-->
  <div data-role="checkbox" class="in-flex wrap rad-r">
    <div checker data-anime="fadeIn" data-assign=":marker2" data-color="@red|lime" class="in-flex rad-r">
    
      <div marker class="b-red"></div>
      <div marker class="b-lime"></div>
      
    </div>
    <input type="checkbox" checked="false" value="M1">
  </div> 
  
  <!--second checkbox section-->
  <div data-role="checkbox" class="in-flex wrap rad-r">
    <div checker data-anime="fadeIn" data-assign=":marker2" data-color="@red|lime" class="in-flex rad-r">
    
      <div marker class="b-red"></div>
      <div marker class="b-lime"></div>
      
    </div>
    <input type="checkbox" checked="false" value="M2">
  </div>
  
  <!--third checkbox section-->
  <div data-role="checkbox" class="in-flex wrap rad-r">
    <div checker data-anime="fadeIn" data-assign=":marker2" data-color="@red|lime" class="in-flex rad-r">
    
      <div marker class="b-red"></div>
      <div marker class="b-lime"></div>
      
    </div>
    <input type="checkbox" checked="false" value="M3">
  </div>
  
</div>

From the sample above, each custom checkbox is sectionalized using the data-role attribute. This attribute specifies the primary parent group as checkbox-list while it's direct children are specified as checkbox. Once this html structure is defined, the plugin will assumed that all the children checkboxes belong to the parent checkbox-list and this will force the plugin to set up a relationship amongst them based on the type of relationship effect applied. There are basically two types of effects available which are the "radio" effect and the "base" effects. These effects are discussed on the next page.
