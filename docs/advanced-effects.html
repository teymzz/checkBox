CheckBox.js


Advanced Effects


The checkbox plugin support extended relationships between custom checkboxes. This feature can be achieved by using a slighty different html structure. Since this activity involves more than one native checkbox, the custom checkboxes will also be more than one. In order to achieve the radio feature, we need to keep the entire checkboxes within a specified parent element. The parent element is usually defined with an attribute of data-role and the value is usually set as "checkbox-list". Once we place the custom checkbox inside this parent element, we can then proceed to define the type of toggle effect we need to achieve. There are basically two types of toggle effect available which are the "radio" and the "base" effects.

Radio Effect

The radio effect is similar to the radio buttons and it is specified by adding an extra attribute of data-bind with a value of 'radio' into the parent element of the custom boxes. An example is shown below.

sample: single color
Copy 
<div checkbox data-color="red" data-label="toggle" data-size="20px">
  <div marker=""></div>
</div>
<input type="checkbox">
<div data-access="label"></div>

In the sample structure above, the label field will be assigned a color of “red” which is specified through the data-color attribute. This means that when the label text “toggle” is added into the label field, the text will be in red color. The data-color also supports a maximum of two alternate colors. A sample format of this is shown below:

sample: double colors
Copy 
<div class="checkbox-wrapper">
  <div checkbox data-size="20px" data-color="red|lime" data-label="toggle">
    <div marker=""></div>
  </div>
  <input type="checkbox">
  <div data-access="label"></div>
</div>

Let's add some simple css to the html structure above :

Custom css
Copy 
.checkbox-wrapper {
  display: inline-flex;
  gap: .5em   
}

HTML View (Toggle on checkbox)

In the sample structure above, two colors were defined. We can see that only the label color is toggled and this does not affect the marker's background color. The first color is only assigned to the label field when the checkbox is unchecked while the second color is applied when the checkbox is checked. In this way, it becomes easier to set colors for labels. The plugin makes it possible to also extend colors to markers such that both labels and markers can be of the same color type. This color extension is only available for elements that support the use of colors. In order to extend colors to markers, we need to prefix the colors defined with the "@" character symbol. A sample of this is shown below:

sample: extended colors
Copy 
<div  class="checkbox-wrapper" data-label="Smile|Frown">
  <div checkbox data-color="@red|gold" data-label="Smile|Frown">

    <div marker="">
      <i class="bi bi-emoji-frown"></i>
    </div>
    <div marker="">
      <i class="bi bi-emoji-smile-fill"></i>
    </div>
    
  </div>
  <input type="checkbox">
  <div data-access="label"></div>
</div>
