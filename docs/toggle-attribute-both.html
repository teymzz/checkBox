CheckBox.js


Toggle Attributes : both


We can specify that a custom checkbox should assign the value from an attribute into another attribute in both cases when the custom checkbox is checked or unchecked. In this situation, the plugin will first remove the value swiftly before re-adding it in both cases. In order to do this, let's create a sample html structure below.

Sample Html Structure 
<div checkers class="foo" data-switch="bar">
<div marker style="background-color: #594fc8;"></div>
</div>
<input type="checkbox">

From the sample html structure, assuming we need to add the bar value into the class attribute when the checkbox is checked or unchecked, we can easily achieve this using the sample script below :

Global: assign value on checked 
<script>
new CheckBox({
assign: {
    target: '[checkers]',
    attr: ['data-switch', 'class', 'both']
}    
});
</script>

Alternatively, we can do this locally using the data-assign attribute

Local Assign Sample Format 
<div checkers class="foo" data-switch="bar" data-assign="both|data-switch|class">
<div marker style="background-color: #594fc8;"></div>
</div>
<input type="checkbox">

From the sample above, each time the checkbox is checked or unchecked, the bar value will be swiftly removed and re-assigned into the class attribute. To see the effect of this, let's add a custom css below

Custom css 
[checkers] {
width: fit-content;
border-radius: 5px;    
border: solid; 
padding: 4px;    
}

[checkers] > [marker] {
width: 20px;
height: 20px;    
background-color: transparent;
transition: width .5s ease-in-out, 
            background-color .5s ease-in-out;
}

.bar[checkers][checked="unchecked"] > [marker] {
width: 30px;
background-color: red;
}

.bar[checkers][checked="checked"] > [marker] {
width: 50px;
background-color: green;
}

The sample css above was added to give a little width and background color animation effect when the value is toggled in and out of the custom checkbox. The html response is shown below:
