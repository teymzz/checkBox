Toggling Attributes


The toggling of an attribute's value on the custom checkbox element is an integrated feature that is mostly useful for handling animations. This feature defines the value assumed by a specified attribute when the custom checkbox is checked or when it is unchecked. In order to do this, we need to specify a primary source attribute and a secondary destination attribute. The primary attribute is expected to anchor the value that is expected to be toggled while the secondary attribute refers to the attribute whose value is toggled. For example, consider the html structure below :

Sample Html Structure 
<div checkbox source="foo bar" destination="">
  <div marker style="background-color: #594fc8;"></div>
</div>
<input type="checkbox">

From the sample html structure, we have a source and a destination attribute. Assuming that we need to toggle the value of the source into the destination, we can do this globally using the “assign” configuration option. A sample format of this is shown below :

Global Assign Sample Format 
assign: {
  attr: ['source', 'destination', <:modifier:>],
}

From the sample above, the attr key is a subkey of assign configuration key. The first two array values of this subkey respectively points to the name of the source and the destination attributes which have a close relationship. By specifying these attributes, we help the checkbox to know which attributes are closely related. The third array value however is an optional value that specifies the state of the checkbox in which the specified source value is either toggled in or toggled out of the destination attribute. These options are discussed below:

  • checked This option toggles the source value into the destination attribute on the custom checkbox when the checkbox is checked and toggles it out when unchecked. This option is also set as the default option if not specified.
    view sample

  • unchecked This option toggles the source value into the destination attribute on the custom checkbox when the checkbox is unchecked and toggles it out when checked.
    view sample

  • both This defines that the source value should be removed and re-toggled into the destination attribute on the custom checkbox in both cases when the checkbox is checked or unchecked.
    view sample

  • :marker1 This option toggles the source value into the destination attribute on the first custom checkbox marker when the checkbox is checked and toggles it out when unchecked.
    view sample

  • :marker2 This option toggles the source value into the destination attribute on the second custom checkbox marker when the checkbox is unchecked and toggles it out when checked.
    view sample

  • :markers This defines that the source value should be removed and re-toggled into the destination attribute on the custom checkbox markers in both cases when the checkbox is checked or unchecked.
    view sample

Similarly to the global configuration, we can also assign values using the data-assign attribute. The sample html format is shown below:

Local Assign Sample Format 
<div checkbox data-assign="checked|source|destination" source="foo" destination=""></div>
<input type="checkbox">

In the sample above, the data-assign attribute's options are separated by pipe. The first option defines the state of the checkbox upon which the toggle effect is applied while the middle option defines the name of the source attribute where the value is picked from. The last value however, specifies the destination attribute where the value toggle effect is applied. To better understand this, let's assume we have an html structure below :

Sample Html 
<div checkbox class="box" data-class="anime" data-assign="checked|data-class|class"></div>
<div marker></div> 
<div marker></div>
</div>
<input type="checkbox"> <br>

From the sample structure above, the anime value will be toggled into the class attribute without affecting the initial value (i.e box).
