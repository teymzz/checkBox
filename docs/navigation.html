CheckBox.js


Navigation


The checkbox plugin supports navigation buttons for any checkbox list that is uniquely defined. In order to uniquely define a checkbox list, an id attribute must be added to the checkbox list container element. A sample of this is shown below.

HTML 

<div id="checklist-1" data-role="checkbox-list" data-bind="radio">

  <!-- add checkbox groups here -->

</div>

In the sample structure above, the id of the checkbox list is specified as "checklist-1". Once we uniquely specify a checkbox list, we can easily fetch the in-built navigation controller methods available for that checkbox by using the checkList() method. The sample below reveals this

JS 
let checkbox = new CheckBox(true);

checkbox = checkbox.check();

/* fetch controller method of specified checklist */
controllers = checkbox.checkList('checklist-1');

console.log(controllers);

The navigation controller methods available are the prev(), next(), switch(), stop() , disable() and enable(), methods. Other methods include frame, sections(), boxes(), nav() and get() methods. These methods are discussed below:

checkList methods description
prev(interval) This method is used to click on the checkbox that comes before the currently selected checkbox element. If there are no currently selected checkbox in a checkbox list, then it starts from the beginning of the custom checkbox list. The interval if supplied, determines the interval in milliseconds that the prev() method should be called. For example a prev(2000) will run every 2 seconds.
next(interval) This method is used to click on the checkbox that comes after the currently selected checkbox element. If there are no currently selected checkbox in a checkbox list, then it starts from the beginning of the custom checkbox list. Similarly to the prev() method, it can be called multiple times. Hence, it also accepts a time interval argument in milliseconds.
stop() This method is used to stop an iteration (or interval) effect that is initiated either by the prev() or next() methods.
switch(index, option) This is used to select a particular custom checkbox section in a checkbox list using the positional index of that checkbox. The minimum positional index is "1" while the maximum index depends on the total number of custom checkbox sections in the specified checklist. For example, switch(1) will click on the first custom checkbox in a checkbox list. The option argument defines the activity of the switch() method. The available options are "both", "on" and "off" options. The "both" option allows the specified custom box to be toggled upon while the "on" ensures that a specified custom checkbox is checked if not already checked. Similarly, the "off" option will only click on the specified custom box when it is checked so as to ensure that it is kept unchecked. It is important to note that the switch() method will not work for disabled checkboxes.
disable(index) This is used to disable a particular checkbox within a checklist using the positional index of the custom checkbox. The least value of the index starts from "1" and above.
enable(index) This is used to enable a particularly disabled checkbox within a checklist using the positional index of the custom checkbox. The least value of the index starts from "1" and above.
frame This method returns the object of the specific checklist item that is targeted with the checkList() method.
sections(index) This method returns all direct sections of a checklist identified with data-role="checkbox" attribute. The optional index if supplied, returns a specific child that occupies the relative positional index supplied. The lowest acceptable index is "1" which points to the first checkbox section. If no index is supplied, then all sections within a checkList is returned.
boxes(index) This method returns all custom checkboxes of within a checklist. The optional index if supplied, returns a specific child that occupies the relative positional index supplied.
nav(index) This method returns all navigation buttons retrieved from the data-nav attribute. If the index is supplied, the specific navigation button that occupies the specified positional index is returned.
get(index) This method is used to target a specific custom checkbox in a checkList while returning the controller methods for that specified custom checkbox.

  • prev(interval) This method is used to click on the checkbox that comes before the currently selected checkbox element. If there are no currently selected checkbox in a checkbox list, then it starts from the beginning of the custom checkbox list. The interval if supplied, determines the interval in milliseconds that the prev() method should be called. For example a prev(1000) will run every second.
  • next(interval) This method is used to click on the checkbox that comes after the currently selected checkbox element. If there are no currently selected checkbox in a checkbox list, then it starts from the beginning of the custom checkbox list. Similarly to the prev() method, it can be called multiple times. Hence, it also accepts a time interval argument in milliseconds.
  • stop() This method is used to stop an iteration effect that is initiated either by the prev() or next() methods. Unlike other methods, it does not take any argument.
  • switch(index, options) This is used to select a particular custom checkbox section in a checkbox list using the positional index of that checkbox. The positional index runs from a minimum of range of one(1) to a maximum range that depends on the total number of available custom checkbox sections in the specified checkbox list. For example, switch(1) will click on the first custom checkbox in a checkbox list. The options however defines the activity of the switch() method. These options are "both", "on" and "off" options. The "both" will click on the specified custom box irrespective of the checkbox state while the "on" will click on the specified custom box only when it is not checked thereby, keeping it checked. However, the "off" will click on the specified custom box only when it is checked, thereby keeping it unchecked. It is important to note that the "off" method will not work for disabled checkboxes.
  • disable(index) This is used to disable a particular checkbox within a checklist using the positional index of the custom checkbox. The least value of the index starts from "1" and above.
  • enable(index) This is used to enable a particularly disabled checkbox within a checklist using the positional index of the custom checkbox. The least value of the index starts from "1" and above.
  • frame This method returns the object of the specific checklist item that is targeted with the checkList() method.
  • sections(index) This method returns all direct sections of a checklist identified with data-role="checkbox". The optional index if supplied, returns a specific child that occupies the relative positional index supplied.
  • boxes(index) This method returns all custom checkboxes of a checklist. The optional index if supplied, returns a specific child that occupies the relative positional index supplied.
  • nav(index) This method returns all navigation buttons retrieved from the data-nav attribute. If the index is supplied, the specific navigation button that occupies the specified positional index is returned.
  • get(index) This method is used to target specific checkboxes and the controller methods for that specific checkbox is returned.
The example below uses the prev(), next(), stop() and switch() checklist navigation controller methods.

HTML JS 
  
<div id="checklist-1" data-role="checkbox-list" data-bind="radio">

  <!--first checkbox section-->
  <div data-role="checkbox">
    <div checkbox data-color="@#615c988d|lime" data-func="update">
      <div marker></div>
      <div marker></div>
    </div>
    <input type="checkbox">
  </div> 
  
  <!--second checkbox section-->
  <div data-role="checkbox">
    <div checkbox data-color="@#615c988d|lime" data-func="update">
      <div marker></div>
      <div marker></div>
    </div>
    <input type="checkbox">
  </div>
  
  <!--third checkbox section-->
  <div data-role="checkbox">
    <div checkbox data-color="@#615c988d|lime" data-func="update">
      <div marker></div>
      <div marker></div>
    </div>
    <input type="checkbox">
  </div>

</div>  

<!-- add some control buttons -->  
<div class="controls">
  <button id="prev">&#10094;</button>
  <button id="stop">&#9724;</button>
  <button id="next">&#10095;</button>
  <input id="switch" type="number" min="1" max="3" placeholder="switch to">
</div>  


let checkbox = new CheckBox(true);

checkbox = checkbox.check({ flip: true });

/* bind controller buttons & input field */
let prevBtn = document.querySelector('#prev');
let nextBtn = document.querySelector('#next');
let stopBtn = document.querySelector('#stop');
let switchField = document.querySelector('#switch');

/* fetch the checklist field & add event to controllers*/
let checkList1 = checkbox.checkList('checklist-1')

prevBtn.addEventListener('click', () => {
    checkList1.prev(1000) 
})

nextBtn.addEventListener('click', () => {
    checkList1.next(1000) 
})

stopBtn.addEventListener('click', () => {
    checkList1.stop() 
})

switchField.addEventListener('change', () => {
    checkList1.stop() /*stop any previous animation*/ 
    checkList1.switch(this.value) 
})

function update(e) {
  if(e.checked) {
      /*update the input field*/
      switchField.value = e.listId;
    }
}

From the sample above, each controller will affect the response of the custom checkboxes in the checklist. It is important to note that the response of any checklist is dependent upon the value of the data-bind attribute. For example the data-bind attribute's value set on the checklist html structure above is "radio". Another sample below is one which uses the switch() method to control the state of the checkbox.

<!-- add some control buttons -->  
<div class="controls">
  <button id="on" class="switch-btn">on</button>
  <button id="off" class="switch-btn">off</button>
  <input id="switch" type="number" min="1" max="3">
</div>  


let checkbox = new CheckBox(true);

checkbox = checkbox.check({ flip: true });

let switchBtns = document.querySelectorAll('.switch-btn');

switchBtns.forEach( switchBtn => {
  
  let parent = switchBtn.closest('.controls');
  let checkList = switchBtn.closest('[data-role="checkbox-list"]')
  let checkListID = checkList.getAttribute('id')

  switchBtn.addEventListener('click', () => { 
    
    const inputBtn = parent.querySelector('input');
    const boxIndex = parseInt(inputBtn.value) || 0;

    const checkList = checkbox.checkList(checkListID)

    const state = switchBtn.getAttribute('id');
   
    if( ['on','off'].includes(state) ) {

       /* turn on or off the selected box index*/
       checkList.switch(boxIndex, state)

    }

  })

})
